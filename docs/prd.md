# software-bom-admin（Lite）产品需求说明

## 1. 产品概述


| 项        | 说明                                                       |
| -------- | -------------------------------------------------------- |
| **产品名称** | software-bom-admin (Lite Version)                        |
| **核心定位** | 实施现场的「本地软件包暂存」与「MD5 校验中心」，校验通过后转存至 ext-Artifactory。      |
| **目标用户** | 项目实施工程师、二次发布管理员。                                         |
| **规模假设** | BOM 表数量预期小于 1000；不将性能、存储量、安全性作为本 Lite 版的设计约束（功能与数据模型优先）。 |


---

## 2. 核心业务流程

1. **BOM 入库**：用户粘贴原始 BOM（表格/文本），系统按行写入数据库；**不假设 BOM 列名与版式永久不变**。
2. **获取文件**：
  - **it-artifactory**：可自动识别链接，使用 API Key 下载到本地约定目录。  
  - **其他来源**（如网盘）：仅提示用户自行下载并拷贝到服务器约定目录；后续由扫描更新状态。
3. **本地扫描与索引**：定时扫描本地目录，维护**本地文件索引表**（见 §5）；按 MD5 与 BOM 行关联。
4. **校验**：本地实测 MD5 与 BOM 中期望 MD5 比对（期望 MD5 来自 jsonb，见 §4）。
5. **远程同步**：校验通过后，查 ext-Artifactory 是否已有相同 MD5；若有则 **复制** 到当前发布约定的 **版本目录** 目标路径，并将可下载 URI 写入该行 **jsonb**（`ext_url`）；若无则 **从本地已校验文件上传** 至该路径并写入 `**ext_url`**（详见 §7）。
6. **离线/SSH 场景**：对指定 BOM 一键生成 **wget/curl** 命令（原始 URL 或 ext URL），供工程师在无 Web 时拉取文件（见 §9）。

## 3. BOM 数据模型（Free Schema）

### 3.1 单一事实来源

- 每条 BOM **行**以 `**jsonb` 存储整行内容**，作为**唯一事实来源**，不将「下载路径」「MD5」等再拆成独立列复制存储，避免数据不一致。
- 应用在读时从 `jsonb` 解析「下载路径」「MD5」（键名可与当前 Excel/表格约定一致；若变更，在解析层做映射即可）。
- 用户可**整表多次粘贴覆盖**以修正；也允许在 UI 中**直接编辑 jsonb 对应字段**（含手补 MD5）。

### 3.2 入库校验（宽松）

- 系统可检查 jsonb 中是否存在语义上的 **「下载路径」** 与 **「MD5」**（按配置的 key 或兼容多 key）。
- **不符合要求时仅提示（告警），不拒绝入库**。

### 3.3 元数据列（可选，非业务字段复制）

- 除 `bom_row`（jsonb）外，仅保留与「行」无关的元数据即可，例如：`id`、所属 BOM 批次/名称、创建时间等；**不在此重复存储下载 URL、清单 MD5**。
- **ext-Artifactory 转存后的可下载 URI** 仅写入该行 **jsonb** 的约定键（如 `**ext_url`**），**不**为 ext 链接单独增加数据库列表列，与 §3.1 单一事实来源一致。

## 4. 状态与展示（统一状态机）

不再并行维护多套「颜色状态」与「同步状态」两套语义；**对用户展示一套业务状态**，底层实现可映射为枚举，例如：


| 状态（示例）   | 含义                                                                                                |
| -------- | ------------------------------------------------------------------------------------------------- |
| 待处理      | 已入库，尚未完成下载/未发现本地文件                                                                                |
| 待人工下载    | 非 it-artifactory 来源，需用户自行拷贝                                                                       |
| 本地已发现    | 索引中已有对应文件，或 MD5 尚未比对完成                                                                            |
| 校验通过     | 本地实测 MD5 与 BOM 期望 MD5 一致                                                                          |
| 校验失败     | 不一致                                                                                               |
| 已转存（或跳过） | 远端已完成：同 MD5 **已存在**时经 **复制** 至本版本路径并写入 `ext_url`，或 **新上传** 成功并写入 `ext_url`（实现上可区分「复制/跳过上传」与「新上传」） |
| 异常       | 下载/上传/路径等失败                                                                                       |


**本地暂存可丢失**：若对应路径下文件不存在，界面可表述为 **「本地未找到」**（或等价文案），并允许下一轮扫描恢复。

**重命名**：若本地采用统一重命名规则，可在 **该行的 jsonb 内**增加字段（如 `local_stored_name`）记录重命名后的文件名，避免仅依赖模糊匹配；仍以 **不重复复制「清单 MD5 / URL」到独立列** 为原则。

## 5. 本地存储与命名

### 5.1 目录

- 使用服务器**单一扁平目录**（如 `/data/bom_store/`），作为暂存库，**可被清空或缺文件**。

### 5.2 命名规范（解决同包名不同架构）

建议格式：

`{baseName}__{arch}__{shortMd5}.{ext}`

- `baseName`：原文件名去扩展名（做安全清洗）。
- `arch`：从 jsonb 取「硬件平台」等字段；取不到可用 `unknown`。
- `shortMd5`：实测 MD5 前 8 位；尚未算出时可占位或待算后再重命名（策略在实现时二选一并写死）。
- `ext`：保留原扩展名（含 `.tar.gz` 等多段后缀）。

**目的**：同名不同架构、不同内容通过 **架构段 + 哈希段** 区分，避免互相覆盖。

## 6. 本地文件索引表（简化）

多份 BOM 可能指向**同一物理文件**；索引表描述「磁盘上有什么」，与 BOM 行通过 **MD5** 对齐。

**表名建议：`local_file`**


| 字段      | 说明                         |
| ------- | -------------------------- |
| `path`  | 相对仓库根的路径（或统一用绝对路径，二选一整站固定） |
| `size`  | 文件大小（字节）                   |
| `mtime` | 文件修改时间                     |
| `md5`   | 内容 MD5（计算完成后写入）            |


主键：`(path)` 或 `id` + `path` 唯一。

扫描策略：小体量下可全量扫描；仅当 **size/mtime 变化**或**新文件**时再重算 MD5 以节省算力（实现细节自定）。文件消失则删除对应行或标记删除（二选一，保持简单即可）。

## 7. Ext-Artifactory 同步与目录结构

### 7.1 配置

- **Base URL**、**API Key**（或 Token）：仅服务端可见（环境变量或等价），**不**暴露给前端、**不**写入导出脚本明文（见 §9.2）。
- **目标 repo**：可在**设置**中配置或与部署环境绑定，与 Lite 单部署场景一致。

### 7.2 ext 链接写入约定

- **仅**在 BOM 行 **jsonb** 内写入 ext 可下载 URI，约定键名 `**ext_url`**（整站统一，或等价键名一处定稿）；**不**为 ext 链接增加独立列表列（与 §3.3 一致）。

### 7.3 同步前置条件与本地无文件

- **仅当** **本地校验通过**（实测 MD5 与 BOM 期望 MD5 一致）后，才执行本节查重与同步，避免将不可信文件推上远端。
- 若本地无对应文件：界面为 **「本地未找到」** 等（见 §4）；用户可对 **支持的链接**（如 it-artifactory）点击 **下载**，由 **worker** 拉取至约定目录，再经 **扫描 → 校验**。**未校验通过前不得**执行本节同步。其他来源仍提示人工拷贝后依赖扫描更新。

### 7.4 远端目录结构（版本树可浏览）

- 制品在 ext 中的目标路径须落在 **当前发布版本** 之下，以便在 Artifactory 中 **按版本浏览时目录完整**，并可配合后续「整版本」批量下载能力（具体打包方式依赖实例能力或 §9 导出）。推荐形态示例：

`{repo}/{version}/{批次或场景}/{模块}/{存储用文件名}`

- **不要**仅用「组件名 + 文件名」在缺乏区分信息时作为全局唯一依据，以免错误覆盖。**存储用文件名**建议遵循 **§5.2**（含 `baseName`、`arch`、`shortMd5` 等），从命名上区分同名不同架构、不同内容。
- **可选补充形态**（与纯内容寻址仓库习惯一致）：若需强约束「路径即内容」，可额外采用 `{md5 前 2 位}/{完整 32 位 MD5 小写}/` 等分段作为路径的一部分或与版本树组合；Lite 版以 **版本树 + §5.2 文件名** 为默认表述，具体以实现定稿为准。

### 7.5 上传前查重与两分支

1. 调用 Artifactory **Checksum Search**（按实际内容的 MD5，接口路径以环境为准，如 `api/search/checksum?md5=...`）。
2. **若 ext 中已存在该 MD5**
  - **不**重新上传字节流。  
  - 调用 Artifactory **Copy**（或等价 REST）将命中制品 **复制到 §7.4 约定的版本目录目标路径**（在常见部署下可为廉价/逻辑复制，不重复占用物理存储）。  
  - 将 **该目标路径对应的可下载 URI** 写回该行 jsonb 的 `**ext_url`**。  
  - 业务状态上体现为 **「已转存（复制 / 跳过上传）」**（与 §4 一致，可与「新上传」区分展示）。
3. **若 ext 中不存在该 MD5**
  - 从 **本地已校验文件**（通过 MD5 与 `local_file` 或约定路径关联）以 **流式读取** 方式 **Deploy/PUT** 至 §7.4 目标路径；**不要求**在本地已有正确文件时再从公网重复下载。  
  - 成功后写回 `**ext_url`**。  
  - **可选**：上传后再做一次 Checksum Search 或接口校验，确认远端与本地 MD5 一致。

### 7.6 Checksum Search 多命中

- 同一 MD5 可能对应 **多条路径**（例如历史版本目录、多次复制所致）。  
- **默认策略**：在返回列表中取 **第一条** 作为 Copy 源（内容相同，任选源路径均可）。  
- 若接口 **不保证结果顺序**，实现上可对候选 **按 URI 字典序排序** 后再取第一条，以保证多次运行 **确定性**。

### 7.7 失败与幂等

- **Copy** 或 **Deploy** 失败：进入 **异常**（或配合重试策略），原因可追溯。  
- **重复触发**同步：目标已存在且与预期内容一致时，宜 **幂等**（成功或 no-op），避免误报失败。

## 8. 页面与任务（摘要）

- **扫描间隔**：用户可配置（如 1–60 分钟）；展示「距离下次扫描剩余时间」；提供 **手动触发扫描**。
- **告警列表**：展示缺少下载路径、MD5 非法或缺失等**非阻断**提示。
- **本地未找到时的下载**：对 **支持的链接**（如 it-artifactory），提供 **下载** 操作，由 **worker** 拉取至约定目录，再经扫描与校验；**未校验通过前不执行 §7 同步**（见 §7.3）。
- **一键生成下载命令**：对指定 BOM（或勾选行）导出 Linux **wget/curl** 脚本/剪贴板，支持 **原始链接** 与 **ext-Artifactory** 两种来源（见 §9）。

## 9. 一键生成下载命令（Linux wget / curl）

面向**指定 BOM**（可选：整张表或勾选多行），提供 **一键导出** 可在 Linux 终端执行的下载片段，支持两类 URL 来源（用户二选一或分别导出两段脚本）。

### 9.1 功能范围


| 来源                  | 说明                                                                                             |
| ------------------- | ---------------------------------------------------------------------------------------------- |
| **原始链接**            | 从 jsonb 解析出的「下载路径」等字段，**不区分**是否为 it-artifactory、网盘直链或其它 HTTPS；按行生成 `wget` 或 `curl` 命令。         |
| **Ext-Artifactory** | 从 jsonb 中已记录的 **ext 转存 URI**（同步后写入的 `**ext_url`**）生成命令；若某行尚未转存成功，该行在导出结果中**跳过并提示原因**，或导出为注释占位。 |


- 支持切换 **wget** / **curl** 风格（或同时生成两种，由用户勾选）。
- 支持指定**输出目录**（如 `-O` 目标路径、`wget -P`），与 §5 本地扁平目录、命名规则对齐时可选用「建议文件名」（含 `__arch__shortMd5` 规则），避免覆盖。

### 9.2 鉴权原则（不在命令中硬编码密钥）

不同站点鉴权方式差异大，PRD 要求**统一安全策略**，避免把 API Key / 密码写进可被提交的脚本文件。

1. **不在导出的 `.sh` 或剪贴板内容中写入明文 Token/密码**（除非用户显式勾选「我接受风险」并仅用于本机一次性使用——Lite 版可不做该选项，默认始终占位）。
2. **推荐：环境变量 + 占位说明**
  - 生成的脚本在开头注释或 `echo` 说明：请先 `export` 下列变量，或 `source` 某 `.env`（由运维自备，不随仓库提交）。  
  - 示例（概念）：`curl -fL -H "Authorization: Bearer $EXT_ARTIFACTORY_TOKEN" -o ... "$URL"`  
  - **Artifactory / JFrog** 常见：`Authorization: Bearer <API Key>`，或 Basic（用户名 + API Key/密码）；具体以实际 ext 与 it 环境文档为准。
3. **Artifactory 直链若支持 URL 内嵌 Basic（不推荐）**：不在 PRD 中作为默认方案；若用户环境仅支持 Basic，可在高级选项中生成 `-u "$USER:$PASS"`，仍要求变量来自环境，不写死。
4. **原始链接、无需鉴权的公开 GET**：命令中可无 `-H`，仅 URL + 输出文件。
5. **需 Cookie / 网盘 token / 一次性签名 URL**：系统**无法从 BOM 推断**；该行导出时附带**注释**，提示「请在本机浏览器登录后替换为有效地址」或「手动补全 Header」，不伪造鉴权。
6. **it-artifactory 原始链接**：与 ext 相同，按部署约定使用 **API Key / Token**；生成命令使用 `$IT_ARTIFACTORY_TOKEN`（或统一命名）占位，与 ext 变量可分开配置（不同仓库不同 Key 时便于区分）。

### 9.3 交付形式

- **剪贴板**：多行命令，便于粘贴到服务器执行。  
- **可选**：下载为 `download-bom-<name>.sh`，首行 `#!/usr/bin/env bash`，`set -euo pipefail`（可选），便于审计与重复执行。

### 9.4 与「应用内自动下载」的关系

- 应用内守护进程仍可用 API Key 读配置完成下载；**本章节**面向「无 UI、仅 SSH 到现场机」的场景，与自动下载互补，**鉴权配置来源可一致**（如同读同一份仅服务器可见的配置），但导出脚本仍遵守 §9.2，不把密钥写进文件内容。

## 10. 验收要点（功能）

- 粘贴任意列结构的 BOM，只要入库成功，**均以 jsonb 保留原貌**；关键字段缺失时**有提示但不拒绝**。
- it-artifactory 可自动下载；其他来源有明确指引，拷贝后扫描能更新索引与关联状态。
- 本地索引仅含 path / size / mtime / md5，且多 BOM 可共用同一 MD5 对应文件。
- ext 同步：**先 Checksum Search**；已存在则 **Copy** 至 **当前版本目录** 并写 jsonb `**ext_url`**；不存在则从 **本地已校验文件** 上传并写 `**ext_url`**；**ext 链接仅存在于 jsonb**，不单独列表列。多命中按 §7.6。版本目录下可浏览到对应制品；**§5.2 存储用文件名** 与校验流程配合，**不因同名覆盖错误文件**。
- 对指定 BOM 可一键生成 **wget/curl** 片段：**原始链接**与 **ext 链接**可区分导出；鉴权通过**环境变量占位**说明，**默认不硬编码密钥**；无 ext 的行有明确跳过或占位说明。
